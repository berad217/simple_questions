# SPEC: High-Uncertainty Cognitive Style Communication Framework

## Project Mission

Create a practical communication framework for individuals who experience persistent high-dimensional uncertainty when answering questions, and for people who communicate with them. This document provides structured dimensionality reduction through question design, response protocols, and shared communication frameworks.

## Target Audiences

1. **People with high-uncertainty cognitive style** - who struggle to answer "simple" questions
2. **People who interact with high-uncertainty individuals** - partners, colleagues, friends
3. **Both together** - relationship-specific communication protocols

#### Section 4.1: Core Techniques (from original document)

**Status**: Complete (section-04-1-core-techniques.md)

**Success Criteria**: Reader can:

- Use all six techniques in next conversation

**Content Requirements**:
For each technique (Domain-binding, Uncertainty permission, Forced-slice, Counterfactual anchoring, Output-format constraints, Role-locking):

- Clear definition
- Mechanism explanation (WHY it reduces cognitive load)
- 3-5 concrete examples across different contexts (professional, intimate, casual)
- When to use this technique vs others
- Common failure modes and corrections

**Template for Each Technique**:

```
**Technique Name**: Brief definition
**Mechanism**: Why this reduces dimensionality
**Examples**:
  - Professional context: [before/after]
  - Intimate context: [before/after]
  - Casual context: [before/after]
**When to Use**: Specific situations where this works best
**Failure Modes**: What goes wrong and how to correct
```

**Key Concepts to Establish**:

- "Reduce dimensionality" not "reduce intelligence"
- Techniques specify WHICH uncertainties to set aside temporarily
- Goal is communicable stability, not perfect truth
