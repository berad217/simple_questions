## Key Decisions Made

**1. Spiral Development over Linear**
- Writing order: Phase 1 (foundation) → Phase 2 (practical) → Phase 3 (theory) → Phase 4 (specialized) → Phase 5 (polish)
- Rationale: Establish voice early, make useful before theoretical, theory can reference concrete examples
- See SPEC.md "Writing Priority Order" section

**2. Success Criteria as Definition of Done**
- Each section measured by "reader can DO X afterward," not word count
- Example: Section 1 success = "reader can recognize this pattern in themselves or others"
- Forces practical utility

**3. Non-Pathologizing Frame Throughout**
- This is a cognitive style with trade-offs, not a disorder to fix
- Key phrase: "amputates uncertainty" (captures why forced collapse feels dishonest)
- Must maintain this frame in every section

**4. Five Core Insights Must Permeate Document**
From genesis_document.md:
- "Amputates uncertainty" - why forced collapse feels dishonest
- "Structured collapse not simpler thinking" - solution approach
- "Reduce dimensionality" - what techniques actually do
- "Lossy compression is not lying" - reframe communication
- "Communicable stability not perfect truth" - goal of framework

**5. Minimal Viable Workflow Documented**
- Not everyone needs full 5-document foundation
- Start with 3 files (outline, log, style), scale up as needed
- This makes workflow accessible to smaller projects

1. Literal information (explicit content)
2. Relationship signal (trust, priority, care)
